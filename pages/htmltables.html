HTML Tables

When should I use a table?

HTML tables are a gridlike structure featuring rows and columns. They are useful for displaying tabular data. Although historically used to create layouts of many webpages, they are not designed for that purpose. In addition to creating a 'tag soup' when attempting to layout a page with table elements they also should be avoided for the following reasons:

  • Tables may impact accessibility

    Tables are difficult for screen readers to parse. Most screen readers will read content from left to right, top to bottom. This may make sense visually with the proper context, but may not work for the overall understanding of a webpage.

  • Layout changes are difficult

    The cumbersome nature of tables may make it difficult to maintain or modify a webpage. The nested tags and possibly nested tables may make it very difficult to understand where an element is or what will happen if you move an element.

  • Tables are not responsive

    Tables can be designed using percentages to be somewhat flexible as a page changes sizes, but generally it is not very responsive. To achieve even moderate responsiveness generally would require much more work than using true layout oriented design techniques.
